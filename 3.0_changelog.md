# XCX 3.0 ‚Äî Changelog (2.2 ‚Üí 3.0)
**April 2026 ¬∑ Type: ANNOUNCEMENT**

> **Note:** XCX 3.0 may contain bugs ‚Äî the VM architecture has grown complex over time. Please report issues. XCX 4.0 is planned with a fully redesigned architecture.

---

## Database (NEW)

Entirely new subsystem. No equivalent existed in 2.2.

- **[NEW]** `database:` block ‚Äî declares a SQLite connection with `engine`, `path`, `timeout`, `readonly` fields
- **[NEW]** Connections are lazy ‚Äî opened on first use; SQLite creates the file automatically
- **[NEW]** Multiple simultaneous connections supported
- **[NEW]** Column attributes for `table:` declarations used by the DB module: `@pk`, `@unique`, `@optional`, `@default(v)`, `@fk(t.col)` ‚Äî `@auto` already existed in 2.2
- **[NEW]** Write operations: `db.push()`, `db.save()` (upsert, requires `@pk`), `db.insert()`
- **[NEW]** Read operations: `db.fetch()`, `db.fetch().where()` (compiles to SQL `WHERE`), `db.query()`, `db.queryRaw()`, `.first()` on raw results
- **[NEW]** Delete operations: `db.remove().where()` (`.where()` required ‚Äî compile error D401 otherwise), `db.truncate()`, `db.exec()`
- **[NEW]** Transactions: `db.begin()`, `db.commit()`, `db.rollback()` ‚Äî auto rollback on `halt.error`
- **[NEW]** Result capture via `as`: returns object with `affected` and `insertId` fields
- **[NEW]** DDL: `db.sync()`, `db.drop()`, `db.has()`
- **[NEW]** `@optional` columns: SQL `NULL` maps to the XCX type's default value on read (`0`, `0.0`, `""`, `false`)
- **[NEW]** `save()` without `@pk` = compile error; `remove()` without `.where()` = compile error (D401)
- **[NEW]** Workaround documented for fiber-local variable scoping bug on Windows (planned fix in 4.0)

## Named Arguments (NEW)

- **[NEW]** `table.add()`, `table.insert()`, and `db.insert()` now accept arguments by column name
- **[NEW]** Positional and named arguments can be mixed ‚Äî positional must come first
- **[NEW]** `@auto` columns cannot be passed explicitly (compile error)
- **[NEW]** Duplicate column names in one call = compile error
- **[NEW]** Namespace separation: left side of `=` is the column name, right side is a local expression ‚Äî no conflict

---

## Type System

- **[NEW]** Type aliases added to simple types table: `i / int`, `f / float`, `s / str`, `b / bool`
- **[NEW]** `array:json` ‚Äî array of JSON objects, documented as a distinct type
- **[NEW]** `database:` added as a complex type
- **[CHG]** Strict collection validation ‚Äî type mismatches in array/set/table literals are now compile errors (e.g. `array:s: names {"Alice", 2}` ‚Üí ERROR [S103])
- **[INT]** Internal "Other" type removed from the type checker

---

## Error System (NEW)

No structured error codes existed in 2.2.

- **[NEW]** Sxxx / Dxxx error code series, consistent across REPL and compiler:

| Code | Description           |
|------|-----------------------|
| S103 | Type mismatch         |
| S108 | Invalid index         |
| S109 | Property not found    |
| S110 | Method not found      |
| S111 | Invalid arguments     |
| S211 | Void fiber            |
| S212 | Invalid fiber run     |
| S302 | Table row mismatch    |

- **[NEW]** `.toStr()` ‚Äî serializes array to a JSON-formatted string
- **[NEW]** `.toJson()` ‚Äî converts array to a native `json` value

## Collections ‚Äî Maps

- **[NEW]** `.toStr()` ‚Äî serializes map to a JSON-formatted string
- **[NEW]** `.toJson()` ‚Äî serializes map to a JSON object; keys converted to strings

## Collections ‚Äî Tables

- **[NEW]** `.toJson()` ‚Äî serializes all rows to a JSON array of objects; `@auto` columns included
- **[NEW]** Named arguments for `.add()` and `.insert()` (see Named Arguments section)
- **[NEW]** Column attributes `@pk`, `@unique`, `@optional`, `@default(v)`, `@fk(t.col)` ‚Äî used when connecting a table to a database
- **[CHG]** Stricter row and column validation

## Collections ‚Äî Sets / random.choice

- **[CHG]** `random.choice from` now works on **both sets and arrays** ‚Äî in 2.2 it was restricted to sets only
- **[CHG]** Picking from an empty set or array now returns `false` (previously undocumented)

- **[NEW]** `crypto.hash(data, "base64_decode")` ‚Äî decodes from Base64

- **[NEW]** `store.isDir(p)` ‚Äî checks if path is a directory
- **[NEW]** `store.size(p)` ‚Äî returns file size in bytes

- **[NEW]** `store.glob(pat)` ‚Äî returns files matching a glob pattern
- **[NEW]** `store.zip(src, target)` ‚Äî archives to zip
- **[NEW]** `store.unzip(zip, dest)` ‚Äî extracts zip

## Standard Library ‚Äî random

- **[NEW]** `random.int(min, max)` / `random.int(min, max @step n)` ‚Äî random integer in range
- **[NEW]** `random.float(min, max)` / `random.float(min, max @step n)` ‚Äî random float in range
- **[CHG]** `random.choice from` now supports arrays in addition to sets (see Collections ‚Äî Sets)

## Terminal & Input

- **[NEW]** `input` module: `input.key()`, `input.key() @wait`, `input.ready()`
- **[NEW]** Full set of key constants: `"UP"`, `"DOWN"`, `"LEFT"`, `"RIGHT"`, `"ENTER"`, `"ESC"`, `"BACKSPACE"`, `"TAB"`, `"F1"`‚Äì`"F12"`, `"KEY_CTRL_C"`, `"KEY_CTRL_Z"`, `"KEY_CTRL_S"`
- **[NEW]** `.terminal !raw` ‚Äî raw mode (no echo, no buffering)
- **[NEW]** `.terminal !normal` ‚Äî restores normal terminal mode
- **[NEW]** `.terminal !cursor on` / `.terminal !cursor off`
- **[NEW]** `.terminal !move x y` ‚Äî moves cursor to position
- **[NEW]** `.terminal !write expr` ‚Äî prints without trailing newline
- **[CHG]** 2.2 had only 3 terminal directives (`!clear`, `!exit`, `!run`); 3.0 adds 6 more

## I/O ‚Äî User Input (`>?`)

- **[CHG]** In 2.2, entering a value of the wrong type caused the variable's type to change at runtime (dynamic typing). In 3.0 this behavior is removed ‚Äî `>?` strictly attempts to parse into the declared type.

---

## JSON

- **[NEW]** `.push(val)` ‚Äî appends a `json` element to a JSON array node; `halt.error` if called on an object
- **[NEW]** `.first()` ‚Äî returns the first element of a JSON array; `halt.error` if empty
- **[NEW]** `.toStr()` ‚Äî renamed from `.to_str()` (see Breaking Changes)
- **[NEW]** Serialization from collections: `.toJson()` available on maps and tables, producing JSON objects and arrays respectively

---

## HTTP ‚Äî Response Object

2.2 response had four fields. 3.0 adds two:

- **[NEW]** `text` ‚Äî raw response body as a string (always available)
- **[NEW]** `error` ‚Äî error message string on request failure; empty string if OK

---

## HTTP ‚Äî CORS

- **[CHG]** Behavior changed from 2.2 to 3.0:
  - **2.2**: Engine added `Access-Control-Allow-Origin: *` automatically; manually adding it caused a browser duplicate header error
  - **3.0**: Engine adds CORS headers only if they are **missing** from the response; explicitly declaring them in code is now **recommended**

- **[NEW]** `.split(separator)` ‚Äî splits string by separator, returns `array:s`

## Breaking Changes

| Area | 2.2 | 3.0 |
|------|-----|-----|
| JSON method | `.to_str()` | `.toStr()` (renamed) |
| `>?` input | Variable type changes at runtime if input mismatches | Strict parsing only, no type change |
| CORS | Do NOT add `Access-Control-Allow-Origin` manually | Recommended to declare headers explicitly |
| `random.choice from` | Sets only | Sets and arrays |
